# Offery — 品牌設計白皮書

**版本**：v1.0
**日期**：2026-04-11
**設計風格**：Calm & Trustworthy — 沈穩信賴與微擬物風（Subtle Neumorphism）

---

## 一、品牌核心

### 1.1 品牌名稱

**Offery**（求職獵人）

### 1.2 品牌標語

> 「找到理想工作的最短路徑。」

### 1.3 品牌使命

為台灣求職者打造最直覺、最有溫度的 AI 求職管理工具，讓每一次投遞都更有信心。

### 1.4 品牌願景

成為繁體中文圈最受信賴的求職管理平台，陪伴每一位求職者從迷茫到從容。

### 1.5 品牌個性（5 大特質）

| 特質 | 說明 |
|------|------|
| **沈穩** | 不浮誇、不焦慮，用專業感給求職者安全感 |
| **信賴** | 穩定可靠，資料安全，像一位值得信任的求職教練 |
| **溫暖** | 不冰冷，帶有人情味的設計語言，讓求職過程少一點壓力 |
| **專注** | 介面簡潔不雜亂，引導使用者專注在最重要的事情上 |
| **智慧** | AI 輔助但不喧賓奪主，工具感強但不複雜 |

## 二、色彩計畫

### 2.1 色彩原則

採用**低飽和度、高明度**的配色策略，營造沈穩信賴的視覺氛圍。

## 四、微擬物風格系統（Subtle Neumorphism）

### 4.1 設計哲學

介於完全平面化（Flat Design）與傳統擬物風之間的中間地帶。透過**柔和的外陰影**與**微妙的內發光**，讓 UI 元素有輕微的立體浮凸感，但不會過度裝飾。

核心精神：
- **克制**：陰影淺而柔，不做誇張凹凸
- **一致**：所有元素共享同一組光源方向（左上 → 右下）
- **功能導向**：立體感服務於可用性，不是純裝飾

### 4.2 卡片樣式

```
┌─────────────────────────────┐
│                             │  ← 圓角 16px (rounded-2xl)
│    ☁ 卡片內容               │  ← 背景 #FFFFFF
│                             │  ← 外陰影：柔和雙層
│                             │  ← 內邊距 24px (p-6)
└─────────────────────────────┘
```

**CSS 實作**：

```css
/* 標準卡片 */
.card-neu {
  background: #FFFFFF;
  border-radius: 16px;
  box-shadow:
    6px 6px 16px rgba(174, 174, 192, 0.2),
    -4px -4px 12px rgba(255, 255, 255, 0.7);
}

/* 卡片 Hover 效果 */
.card-neu:hover {
  box-shadow:
    8px 8px 20px rgba(174, 174, 192, 0.25),
    -6px -6px 16px rgba(255, 255, 255, 0.8);
  transform: translateY(-1px);
}

/* 輸入框（微凹陷） */
.input-neu {
  background: #FCFBF9;
  border-radius: 12px;
  box-shadow:
    inset 2px 2px 5px rgba(174, 174, 192, 0.15),
    inset -2px -2px 5px rgba(255, 255, 255, 0.5);
}
```

### 4.3 Tailwind CSS 自定義

```css
/* globals.css 中加入 */
@layer utilities {
  .shadow-neu {
    box-shadow:
      6px 6px 16px rgba(174, 174, 192, 0.2),
      -4px -4px 12px rgba(255, 255, 255, 0.7);
  }
  .shadow-neu-hover {
    box-shadow:
      8px 8px 20px rgba(174, 174, 192, 0.25),
      -6px -6px 16px rgba(255, 255, 255, 0.8);
  }
  .shadow-neu-inset {
    box-shadow:
      inset 2px 2px 5px rgba(174, 174, 192, 0.15),
      inset -2px -2px 5px rgba(255, 255, 255, 0.5);
  }
  .shadow-neu-pressed {
    box-shadow:
      inset 3px 3px 8px rgba(174, 174, 192, 0.25),
      inset -3px -3px 8px rgba(255, 255, 255, 0.6);
  }
}
```

## 五、按鈕與互動元素

### 5.1 按鈕體系

| 類型 | 外觀 | 用途 |
|------|------|------|
| **主要按鈕** | 鼠尾草綠底 + 白字 + 微擬物陰影 | CTA、主要操作（儲存、新增） |
| **次要按鈕** | 白底 + 鼠尾草綠框線 + 鼠尾草綠字 | 次要操作（取消、返回） |
| **文字按鈕** | 無邊框 + 鼠尾草綠字 | 連結式操作（了解更多、查看全部） |
| **危險按鈕** | 柔紅底 + 白字 | 危險操作（刪除帳號、移除職缺） |
| **幽靈按鈕** | 透明底 + 霧灰文字 | 次要操作、密集排列場景 |

### 5.2 按鈕尺寸

| 尺寸 | 高度 | 內邊距 | 字級 | 圓角 |
|------|------|--------|------|------|
| Large | 48px | px-6 py-3 | text-base | rounded-xl |
| Medium | 40px | px-4 py-2.5 | text-sm | rounded-lg |
| Small | 32px | px-3 py-1.5 | text-xs | rounded-md |

### 5.3 按鈕狀態

| 狀態 | 表現 |
|------|------|
| Default | 正常顯示 + 微擬物外陰影 |
| Hover | 顏色加深 5% + 陰影加大 + `translateY(-1px)` |
| Active/Pressed | 微擬物內陰影（凹陷感） |
| Disabled | 透明度 50% + 禁用游標 |
| Loading | 文字替換為 spinner + 禁用互動 |

### 5.4 圖示風格

- **圖示庫**：Lucide React（線條風格，與品牌簡潔感一致）
- **預設尺寸**：20px（h-5 w-5）
- **顏色**：跟隨文字顏色，不額外上色
- **描邊寬度**：1.5px（Lucide 預設）

### 5.5 微動畫原則

| 類型 | 時長 | 曲線 | 說明 |
|------|------|------|------|
| Hover 過渡 | 150ms | ease-out | 按鈕、卡片 hover 效果 |
| 頁面切換 | 200ms | ease-in-out | Route 切換淡入 |
| 展開收合 | 250ms | ease-out | 表單展開、下拉選單 |
| 通知出現 | 300ms | spring | Toast 通知滑入 |

**原則**：動畫要快到不會被注意到「在等待」，但慢到能感受到「流暢」。不使用彈跳、旋轉等花俏動畫。

## 六、畫面佈局

### 6.1 留白哲學

> 「留白不是空，是呼吸的空間。」

- 區塊間距：32px（py-8）以上
- 卡片間距：16px（gap-4）
- 卡片內邊距：24px（p-6）
- 表單欄位間距：16px（space-y-4）

### 6.2 頁面最大寬度

| 頁面類型 | 最大寬度 | 說明 |
|----------|---------|------|
| Landing Page | 1280px（max-w-7xl） | 寬版，展示性強 |
| Dashboard 內容區 | 無限制（flex-1） | 隨側邊欄自適應 |
| 表單頁面 | 640px（max-w-xl） | 聚焦型，減少視覺干擾 |
| 認證頁面 | 448px（max-w-md） | 居中，簡潔 |

### 6.3 響應式斷點

| 斷點 | 寬度 | 佈局調整 |
|------|------|---------|
| Mobile | < 640px | 單欄、側邊欄改為底部 Tab 或漢堡選單 |
| Tablet | 640-1024px | 雙欄、Kanban 改為列表檢視 |
| Desktop | > 1024px | 完整佈局、側邊欄 + 主內容區 |

## 七、語氣與文案風格

### 7.1 語氣光譜

```
冰冷 ←──────────── ★ ──────────→ 過度親暱
          專業但溫暖，像一位好朋友兼職涯教練
```

### 7.2 撰寫原則

| 原則 | 說明 | 範例 |
|------|------|------|
| **鼓勵但不空洞** | 不說「加油」，給具體的下一步 | ✅「你的履歷已完成 80%，加上技能區塊會更完整」 |
| **簡潔直接** | 一句話說完重點，不繞彎 | ✅「新增職缺」 ❌「點擊此按鈕以新增一個新的職缺記錄」 |
| **去焦慮化** | 求職本身壓力大，介面不加壓 | ✅「尚無職缺」 ❌「你還沒有任何職缺！趕快新增！」 |
| **用「你」不用「您」** | 親近但不失分寸 | ✅「歡迎回來」 ❌「歡迎您回來」 |

### 7.3 各情境文案範例

| 情境 | 文案 |
|------|------|
| 空狀態（職缺） | 「尚無追蹤中的職缺。新增你的第一個職缺，開始掌握求職進度。」 |
| 空狀態（履歷） | 「還沒有履歷。建立你的第一份履歷，開始求職之旅。」 |
| 儲存成功 | 「已儲存」 |
| 刪除確認 | 「確定要刪除這個職缺嗎？此操作無法復原。」 |
| 升級提示 | 「升級 Pro 方案，解鎖 AI 履歷優化等完整功能。」 |
| 免費方案上限 | 「你已建立 3 份履歷（免費方案上限）。升級 Pro 方案可建立無限份履歷。」 |
| 歡迎回來 | 「歡迎回來 👋」 |
| 登入錯誤 | 「電子郵件或密碼不正確，請重新輸入。」 |

## 八、設計系統程式化

### 8.1 Theme 常數（`src/lib/theme.ts`）

所有品牌色彩、間距、圓角、陰影均在此檔案常數化，元件一律引用常數，禁止硬編碼色票值。

### 8.2 CSS 變數（`globals.css`）

品牌色彩透過 CSS 變數定義，確保主題一致性並支援未來深色模式擴展。

### 8.3 元件封裝

以下元件應封裝品牌樣式：
- `Button`：主要 / 次要 / 文字 / 危險 四種變體
- `Card`：標準微擬物卡片
- `Input`：微凹陷輸入框
- `Badge`：狀態標籤
- `Toast`：通知提示

## 九、品牌應用規範

### 9.1 Logo 使用規範

- Logo 由 Lucide `Briefcase` 圖示 + 「Offery」文字組成
- 最小尺寸：圖示 20px + 文字 16px
- 安全區域：Logo 四周至少留 Logo 高度的 50% 空白
- 禁止行為：拉伸、旋轉、加邊框、改色（允許白色反轉版）

### 9.2 深色模式（未來規劃）

| 元素 | 淺色模式 | 深色模式 |
|------|---------|---------|
| 背景 | `#FCFBF9` | `#1A1B2E` |

| 主色 | `#688F79` | `#7A9D89` |
| 文字 | `#3A3A3A` | `#E8E8ED` |

> 深色模式為 v2.0 規劃功能，目前僅做淺色模式。
