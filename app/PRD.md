## 一、產品概述

### 產品願景

打造台灣市場第一個繁體中文 AI 求職管理平台，參考 Teal HQ 核心功能，整合職缺追蹤、履歷優化、求職信生成等工具，幫助求職者提升求職效率 58%。

### 目標用戶

- **主要**：台灣求職者找外商公司工作，或是海外求職，需要系統化管理求職流程的專業人士
- **次要**：台灣求職者（應屆畢業生、轉職者、資深工作者）找台灣本土公司的工作

### 核心價值主張

一站式求職管理平台，告別在 Excel、Google Doc、多個求職網站之間反覆切換的痛苦。

### 商業模式

免費 + 訂閱制（Stripe）

- **免費方案**（永久免費）：基本功能
- **Pro 方案**：NT$299/月，解鎖全部 AI 功能

## 三、使用者旅程（User Journey）

### 3.1 首次使用流程（FTUE）

1. 首頁 → 點擊「免費開始使用」
2. 註冊（Email 或 Google OAuth）
3. 進入 Dashboard 總覽頁
4. 引導建立第一份履歷或新增第一個職缺

### 3.2 日常使用場景

**場景 A：職缺追蹤**

> 小安，28 歲，前端工程師，正在積極求職。她在 104、CakeResume、LinkedIn 上看到有興趣的職缺，將職缺資訊複製到 JobHunter，在看板上追蹤每個職缺的投遞狀態。
>
> 此場景驗證的產品價值：集中管理多平台職缺，取代 Excel 手動追蹤。

**場景 B：AI 履歷優化**

> 阿傑，35 歲，後端工程師，想轉職到外商。他將目標職缺描述貼入 JobHunter，系統自動比對他的履歷，找出遺漏的關鍵字並建議優化方向。
>
> 此場景驗證的產品價值：AI 輔助提升履歷-職缺匹配度，增加面試邀約率。

### 3.3 關鍵旅程指標

| 指標                | 目標     |
| ------------------- | -------- |
| 註冊 → 首次新增職缺 | < 3 分鐘 |
| 每週活躍使用天數    | ≥ 3 天   |
| 免費 → Pro 轉換率   | ≥ 5%     |

## 五、資料模型

| 資料表             | 說明                     | RLS                  |
| ------------------ | ------------------------ | -------------------- |
| `profiles`         | 使用者個人資料、訂閱狀態 | 使用者只能讀寫自己的 |
| `resumes`          | 履歷（JSON 結構化內容）  | 使用者只能讀寫自己的 |

| `contacts`         | 人脈管理                 | 使用者只能讀寫自己的 |

### 安全輔助函式（SECURITY DEFINER）

| 函式                          | 用途                       |
| ----------------------------- | -------------------------- |
| `check_resume_limit(user_id)` | 免費方案履歷上限 3 份      |
| `handle_new_user()`           | 新用戶註冊自動建立 profile |
| `handle_updated_at()`         | 自動更新 updated_at 時間戳 |

## 九、訂閱制架構

### 架構原則：DB 為唯一真相來源

```
用戶購買 → Stripe Checkout → Webhook → API Route → Supabase DB 更新
                                                        ↓
App 載入 → 讀取 DB subscription_tier → 判斷功能權限
```

### 訂閱狀態值

| 狀態   | 說明                     |
| ------ | ------------------------ |
| `free` | 免費用戶                 |
| `pro`  | 訂閱中（含所有 AI 功能） |

### 免費方案限制（DB 層 SECURITY DEFINER 強制）

## 十、已決策問題

| 決策         | 結果                    | 理由                                             |
| ------------ | ----------------------- | ------------------------------------------------ |
| 支付方案     | Stripe 訂閱制           | 全球化支援、API 完善、支援台灣信用卡             |
| 狀態管理     | Zustand                 | 輕量、TypeScript 友好、無 boilerplate            |
| 資料驗證     | Zod                     | 型別安全 + runtime 驗證                          |
| 訂閱狀態來源 | Supabase DB 唯一真相    | 避免 Stripe SDK + DB 不一致（參考 WedSnap 經驗） |
| 數量限制實作 | DB 層 SECURITY DEFINER  | 前端檢查可被繞過（參考 WedSnap 經驗）            |
| 部署平台     | Vercel                  | Next.js 原生支援、免費額度充足                   |
| 設計風格     | Calm & Trustworthy（微擬物風） | 沈穩信賴，低飽和靛藍主色 + 暖奶白背景（詳見 BRAND_GUIDELINES.md） |
